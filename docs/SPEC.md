# 출석 규칙 명세

스터디 공지문을 봇이 구현 가능한 규칙으로 옮긴 것. 애매한 부분은 "결정"으로 표시하고 근거를 남긴다.

## 1. 시간대 (KST 기준)

| 시간대 | 시작 | 종료 | 이모지 |
|---|---|---|---|
| 오전 | 09:00 | 12:00 | 🌅 |
| 저녁 | 19:00 | 22:00 | 🌙 |

- 모든 시간 판정은 **Asia/Seoul** 기준. 서버(OCI VM)의 시스템 시간대와 무관하게 코드에서 KST로 계산한다.

## 2. 출근 / 퇴근

- **출근**: 해당 시간대에 스터디 음성채널에 **최초로 입장**한 시각
  - 공지 형식: `🌅 홍길동님 오전 출근 (09:02)`
  - 시간대 시작 **전부터** 접속해 있던 경우 → 시간대 시작 시각(09:00)을 출근으로 본다
- **퇴근**: 해당 시간대의 **마지막 퇴장** 시각. 공지는 **디바운스 방식** (결정: 실시간감과 도배 방지의 절충)
  - 퇴장 후 **10분간 재입장이 없으면** 그 시점에 퇴근 공지 (퇴장 시각 기준)
  - 10분 내 재입장하면 공지 없음 — 잠깐 자리 비운 것으로 취급
  - 퇴근 공지가 나간 뒤 재입장하면 `재입장` 공지. 이후 다시 퇴장하면 새 퇴근 공지가 나간다 (마지막 퇴근 공지가 최종)
  - 시간대 종료(12:00/22:00)까지 접속 중인 경우 → 종료 시각이 퇴근. 정산 시점에 공지
  - **공지는 실시간이지만 출석 기록·판정은 정산(12:00/22:00)에서만** 한다 — 퇴근 공지의 ✅/⚠️ 표시는 그 시점 누적 기준의 안내일 뿐, DB 기록은 정산이 담당

## 3. 출석 인정

- 시간대 내 **누적 참여 시간 ≥ 60분**이면 출석 1회 인정 (2026-07-06 완화: 120→60분 — 진입장벽은 낮추고 경쟁은 랭킹의 누적 시간이 담당)
  - 들락날락한 경우 모든 구간을 **합산**한다
  - 시간대 밖 참여(08:30 입장 등)는 시간대 경계로 **잘라서(clip)** 계산한다
    - 예: 08:30 ~ 11:30 접속 → 09:00 ~ 11:30 = 150분 → 인정
    - 예: 11:00 ~ 12:40 접속 → 11:00 ~ 12:00 = 60분 → 인정 (경계 밖 40분은 미포함)
    - 예: 11:30 ~ 12:40 접속 → 11:30 ~ 12:00 = 30분 → 미달
- **같은 날 오전·저녁 모두 참여해도 출석은 1회** (공지문 명시)
- 기준(1시간) 미만 참여는 출석으로 인정하지 않되, 퇴근 공지에 누적 시간을 표시해 본인이 알 수 있게 한다

## 4. 주간 점검 · 경고 · 추방

- **주기**: 매주 **일요일 22:05 (KST)** — 저녁 정산(22:00) 직후
  - (결정: 일요일 22:00 정산이 끝나면 그 주 출석은 더 변하지 않는다 — 사실상 "주 마감" 시점. 매일 검사하지 않는 이유: 같은 미달 상태로 연속 경고가 쌓이는 문제)
- **기준**: **이번 주(월~일) 출석 일수 < 3** → 경고 +1
- **추방**: 경고 누적 **3회** 도달 시 자동 Kick + 공지
- **점검 제외 대상**:
  - 봇 계정
  - 운영진 (서버 관리 권한 보유자)
  - **신규 멤버** — 가입 후 7일의 적응 기간
  - **유예 등록 멤버** (아래 "사정 있는 멤버 유예" 참고)
- **면제 판정 원칙 (신입·유예 공통)**: 면제 기간이 **점검 주(월~일)에 하루라도 걸쳐 있으면 그 주는 통째로 면제**된다. 점검은 "복귀(가입) 후 첫 온전한 주"부터 시작
  - (결정: 토요일에 유예가 끝난 사람이 다음날 일요일 점검에서 "이번 주 출석 0회"로 경고받는 불합리 방지. 유예로 쉰 주의 출석을 요구하지 않는다)
  - 예: 유예가 토요일(7/11)까지 → 그 주 일요일(7/12) 점검 면제 → 다음 주(7/13~19)부터 3회 필요
  - 예: 수요일(7/1) 가입 신입 → 유예 끝 7/8이 둘째 주에 걸침 → 7/12 점검도 면제 → 7/19 점검부터 대상
- 경고는 활동 중에는 자동 소멸하지 않는다 (운영진 수동 조정 기능은 Phase 5에서 검토)

### 사정 있는 멤버 유예 (시험 기간, 여행 등)

부득이한 사정은 봇이 판단하지 않는다 — 본인이 운영진에게 말하면 운영진이 등록한다. 두 방식 모두 지원:

| 방식 | 사용법 | 만료 |
|---|---|---|
| **`/유예` 명령어** (권장) | `/유예 대상:@홍길동 일수:14` → SQLite에 기간 저장 | **자동 만료** — 기간이 지나면 다음 점검부터 자동 포함 |
| **`출석유예` 역할** | 역할을 만들어 멤버에게 부여 | 수동 — 역할을 떼기 전까지 무기한 제외 |

- `/유예`, `/유예해제`는 **멤버 추방 권한 보유자(운영진)에게만 보인다** (`setDefaultMemberPermissions`)
- 유예 중에도 출석 기록은 정상 누적된다 — 점검에서만 빠질 뿐
- 유예는 경고를 지우지 않는다 (점검 일시정지 개념. 복귀하면 기존 경고 수 그대로 이어짐)
- 본인은 `/출석현황`에서 자신의 유예 상태(언제까지)를 확인할 수 있다

### 추방과 재입장 — 운영 철학

> 추방은 처벌이 아니라 **"현재 활동 중인 멤버"로 서버를 유지하기 위한 자리 정리**다.
> 바쁘면 자연스럽게 정리되고, 여유가 생기면 언제든 돌아온다.

- **재입장 허용**: Kick이므로 초대 링크로 언제든 다시 들어올 수 있다 (Ban 아님)
- **서버를 떠나면(자진 퇴장·자동 추방 모두) 경고 0으로 초기화**: 재입장하면 깨끗한 상태로 새 출발
  - (결정 1: 경고를 유지하면 재입장 후 한 주만 미달해도 즉시 재추방 — "다시 오는 문은 열어둔다"는 취지와 모순)
  - (결정 2: 자진 퇴장만 경고를 유지시키는 차등은 실익이 없다 — 어차피 추방이 경고를 리셋하므로 "세탁 방지" 효과가 없고, 오히려 자진해서 나간 사람이 추방당한 사람보다 불리해지는 모순만 생김. 규칙을 하나로 통일)
- **재입장자는 신규 멤버와 동일 취급**: 가입 후 7일 유예 적용 (디스코드가 가입 시각을 재입장 시각으로 갱신하므로 별도 처리 없이 자동 적용)
- **출석 이력은 삭제하지 않는다**: 경고 카운트만 초기화. 출석 기록은 user ID 기준으로 보존 (통계·이력 확인용)

자동 추방(봇)과 직접 추방(운영진)은 성격이 다르므로 수단도 다르게 쓴다:

| 구분 | 주체 | 수단 | 재입장 | 용도 |
|---|---|---|---|---|
| 자동 추방 | 봇 | **Kick** | ✅ 가능 (경고 초기화 후 새 출발) | 출석 미달 — 자리 정리 |
| 직접 추방 | 운영진 | **Ban (차단)** | ❌ 불가 | 문제 행동 등 운영진 판단 |

- Ban은 디스코드 기본 기능(멤버 우클릭 → 차단)이라 봇 개입이 필요 없다. 차단 해제 전까지 어떤 초대 링크로도 재입장 불가
- 한계: Ban은 계정+차단 시점 IP 기준이라, 작정하고 새 계정을 만들면 우회 가능 (소규모 스터디에서는 충분한 수준)
- 봇의 주간 점검은 서버 멤버만 대상으로 하므로 Ban된 유저는 자연히 제외된다

## 5. 엣지 케이스 정리

| 상황 | 처리 |
|---|---|
| 08:50 입장 → 계속 접속 | 09:00 크론이 구간을 열어줌. 출근 시각 = 09:00 |
| 09:30 입장 → 10:00 퇴장 → 10:30 재입장 → 12:00까지 | 구간 합산 30 + 90 = 120분 → 출석 인정. 출근 09:30, 퇴근 12:00 |
| 12:00 넘어서도 계속 접속 | 12:00에 구간을 닫고 정산. 이후 접속은 저녁 시간대에 다시 계산 |
| 오전 2시간 + 저녁 3시간 | 출석 1회 (오전에서 이미 인정, 저녁은 기록만) |
| 시간대 외(예: 15:00) 입장 | 무시 (기록하지 않음) |
| 10:00 퇴장 → 10:05 재입장 | 아무 공지 없음 (10분 내 복귀) — 구간만 기록 |
| 10:00 퇴장 → 재입장 없음 | 10:10에 퇴근 공지 (표기 시각은 10:00) |
| 퇴근 공지 후 11:00 재입장 → 11:20 퇴장 | 재입장 공지 → 11:30에 새 퇴근 공지 (마지막 것이 최종) |
| 11:55 퇴장 (종료 5분 전) | 디바운스 만료(12:05)가 종료를 넘김 → 12:00 정산 때 공지 (퇴근 시각 11:55) |
| 디바운스 대기 중 봇 재시작 | 타이머 소실 — 보류되던 퇴근 공지는 정산 때 보완. 드물게 중복/지연 공지 가능 (감수) |
| 봇이 시간대 중간에 재시작 | 재시작 시점 이전의 열린 구간은 닫힌 것으로 처리하고, 접속 중인 멤버는 재시작 시각부터 새 구간을 연다 (일부 시간 손실 가능 — 감수) |
| 봇이 정산 시각(12:00)에 꺼져 있었음 | 다음 실행 시 미정산 시간대를 감지해 늦게라도 정산 |
| 유예가 토요일에 끝남 | 유예가 이번 주에 걸쳐 있으므로 일요일 점검 면제. 다음 주(첫 온전한 주)부터 점검 |
| 신입의 7일 유예가 주 중간에 끝남 | 그 주 점검도 면제 (걸친 주 원칙). 그다음 주부터 점검 대상 |
| `출석유예` 역할을 점검 직전에 뗌 | 역할은 뗀 시점을 시스템이 모름 → 떼는 즉시 점검 대상. **역할 해제는 일요일 점검(22:05) 이후 권장**, 기간 유예는 `/유예`가 안전 |
| 자동 추방 후 재입장 | 경고 0에서 새 출발 + 신규 멤버 7일 유예. 출석 이력은 보존 |
| 자발적으로 나갔다 재입장 | 자동 추방과 동일 취급 — 경고 0에서 새 출발 + 신규 7일 유예. 출석 이력은 보존 |
| 운영진이 직접 차단(Ban)한 유저 | 디스코드가 재입장을 막음. 봇은 관여하지 않으며 점검 대상에서 자연 제외 |

## 6. 공지 메시지 명세

봇이 보내는 모든 메시지의 확정 형식. 구현 시 이 문구를 그대로 사용한다. (`{이름}`은 서버 별명 우선, 없으면 유저명)

**이름 강조 규칙**:
- 출근/퇴근/재입장 등 일상 메시지 → **굵게**(`**{이름}**님`). 눈에 띄되 알림(핑)은 보내지 않는다.
- 경고 → **멘션**(`<@유저ID>`). 본인이 꼭 봐야 하므로 알림이 가게 한다.
- 추방 → 굵게(대상이 서버를 떠나므로 남은 사람들에게 보이는 안내).

### 출근 (시간대 내 최초 입장 시, 즉시)

```
🌅 {이름}님 오전 출근 (09:02)
🌙 {이름}님 저녁 출근 (19:01)
```

### 퇴근 (퇴장 10분 후 재입장 없을 때, 디바운스)

```
🏁 {이름}님 오전 퇴근 (10:30) — 누적 2시간 41분 ✅ 출석 인정
🏁 {이름}님 오전 퇴근 (10:30) — 누적 0시간 42분 ⚠️ 아직 1시간 미만 (재입장하면 이어서 누적돼요)
```

- 시각은 실제 퇴장 시각(10:30), 공지는 10분 뒤(10:40)에 올라간다
- ✅는 그 시점 누적이 이미 60분 이상이라 번복될 수 없는 경우에만 표시

### 재입장 (퇴근 공지가 나간 뒤 다시 들어온 경우)

```
🔄 {이름}님 오전 재입장 (11:00)
```

- 10분 내 재입장은 아무 공지도 하지 않는다 (퇴근 공지 자체가 안 나감)

### 정산 (12:00 / 22:00 — 아직 퇴근 공지가 안 나간 참여자만)

```
🏁 {이름}님 오전 퇴근 (12:00) — 누적 2시간 41분 ✅ 출석 인정
🏁 {이름}님 저녁 퇴근 (22:00) — 누적 3시간 0분 ✅ 출석 인정 (오늘 오전에 이미 인정됨)
```

- 대상: 종료 시각까지 접속 중이던 사람 + 퇴장 후 10분이 안 지나 공지가 보류 중이던 사람
- 이미 퇴근 공지가 나간 사람은 재공지하지 않는다 (기록만 정산)
- 마지막 줄 케이스: 저녁에 기준을 채웠지만 같은 날 이미 출석 인정된 경우

### 주간 점검 — 경고 (일요일 22:05)

```
⚠️ @{멘션} 경고 1/3 — 이번 주 출석 2회 (기준: 3회 이상)
```

- 경고는 본인이 꼭 봐야 하므로 **멘션**(알림이 감)으로 보낸다

### 주간 점검 — 자동 추방

```
👋 {이름}님이 경고 3회 누적으로 자리 정리되었습니다. 여유가 생기면 언제든 다시 돌아오세요!
```

- 추방 공지와 함께 해당 유저의 경고를 0으로 초기화한다

### /출석현황 (본인에게만 보이는 ephemeral 응답)

```
📊 {이름}님의 출석 현황
- 이번 주(월~일) 출석: 2회 (기준: 3회 이상)
- 일요일까지 1회 더 필요해요
- 경고: 1/3
- 유예 중: 2026-07-20까지 점검 제외 🛌   ← 유예 등록된 경우에만 표시
```

### /유예, /유예해제 (운영진 전용, 실행 채널에 공개 응답)

```
🛌 {이름}님을 {YYYY-MM-DD}까지 출석 점검에서 제외합니다.
유예가 걸친 주는 통째로 면제되고, {재개일 YYYY-MM-DD}(월)부터 점검이 다시 적용돼요.

✅ {이름}님의 출석 유예를 해제했습니다. 다음 주간 점검부터 다시 포함돼요.
```

- 재개일 = 유예 종료일이 걸친 주의 **다음 주 월요일** (봇이 자동 계산해 표시)

### 주간 랭킹 (일요일 22:05, #명예의-전당 — 주간 점검 경고에 앞서 공지)

```
🏆 이번 주 출석 랭킹 (7/6 ~ 7/12)
🥇 🌳 {이름} — 5일 · 14시간 32분
🥈 🌿 {이름} — 4일 · 10시간 5분
🥉 🌱 {이름} — 3일 · 7시간 41분

이번 주도 다들 수고했어요! 🔥
```

- 이름 앞 이모지는 현재 레벨 뱃지. 최대 10명, 현재 멤버만
- **정렬**: 출석일수 → 같으면 누적 공부시간 순
- **공동 순위**: 일수·시간이 모두 같으면 같은 순위 + `(공동)` 표기. 다음 순위는 건너뛴다 (예: 공동 2위 2명 → 다음은 4위)
- 출석 기록이 아무도 없으면 생략

### 월간 랭킹 (매월 1일 00:05, #명예의-전당 — 지난달 1일~말일 집계)

```
👑 2026년 7월 출석 랭킹
🥇 🌳 {이름} — 22일 · 51시간 10분
...

한 달 동안 정말 수고 많았어요! 🎊
```

### 레벨 업 (출석이 새로 인정되어 레벨이 오를 때, #명예의-전당)

```
🎉 {이름}님 레벨 업! → Lv.2 🌿 뿌리내림 (누적 출석 10회)
```

- 레벨 기준(누적 출석): Lv.1 🌱 새싹(1회) / Lv.2 🌿 뿌리내림(10회) / Lv.3 🌳 꾸준나무(25회) / Lv.4 🔥 불꽃(50회) / Lv.5 💎 고인물(100회)
- 레벨 역할이 자동 생성·부여되고 이전 레벨 역할은 회수된다 (멤버 목록에서 레벨 색상 표시)

### 신입 환영 (새 멤버 입장 시, #환영-인사 채널에 멘션)

새 멤버는 위로 스크롤하지 않으므로, 환영 멘션이 **전체 채널 안내글 역할**까지 겸한다 (별도 고정 안내글 없음).
첫 줄은 마크다운 헤더(`#`)로 크게 표시.

```
# 👋 @{멘션}님, 미라클 알고리즘 스터디에 오신 것을 환영합니다!

꾸준한 코딩테스트 학습 습관을 만드는 스터디예요. 아래 순서로 둘러보세요! 🙌

1️⃣ #공지사항 — 스터디 규칙과 출석 방식 (필독!)
2️⃣ #명령어-가이드 — 봇 명령어와 레벨 시스템 안내
3️⃣ #모각코 — 스터디 음성채널. 화면 공유를 켜고 각자 문제를 풀어요 💻
4️⃣ #출석-알림 — 출근/퇴근/출석이 자동으로 기록되는 곳
5️⃣ #명예의-전당 — 주간·월간 출석 랭킹과 레벨업 소식 🏆
6️⃣ #스터디-대화방 — 문제 질문·풀이 토론·오늘 푼 문제 공유는 여기서!

🌱 @{멘션}님의 출석 점검은 {재개일 YYYY-MM-DD}(월)부터 적용돼요. 그 전까지는 편하게 적응하시면 됩니다.
궁금한 점은 운영진에게 편하게 물어봐 주세요! 😊
```

- 재개일 = 가입일+7일이 걸친 주의 다음 주 월요일 (봇이 자동 계산)
- 채널명은 실제 채널 멘션(클릭 이동)으로 전송
- 스터디원 역할 자동 부여와 함께 동작. `WELCOME_CHANNEL_ID` 비우면 비활성화

## 7. 봇이 하지 않는 것

- **화면 공유 강제**: 규칙이지만 자동 제재하지 않음 (Phase 5에서 리마인드 메시지만 검토)
- **경고 사면 판단**: 부득이한 사정의 판단은 사람(운영진)의 몫

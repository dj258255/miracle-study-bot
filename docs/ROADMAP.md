# 개발 로드맵

로컬 PC에서 개발/검증 후 Oracle Cloud(OCI) 무료 VM으로 이전하는 흐름.
각 Phase는 독립적으로 완료 확인이 가능하도록 나눴다.

```
Phase 0 ─ 준비        (30분)   봇 계정 생성, 서버 초대
Phase 1 ─ 뼈대        (1시간)  봇 로그인 + 음성채널 이벤트 수신 확인
Phase 2 ─ 출석 코어    (반나절)  출근/퇴근 자동 기록 + 2시간 정산
Phase 3 ─ 규율 자동화   (반나절)  주간 점검, 경고, 추방, 조회 명령어
Phase 4 ─ OCI 배포    (반나절)  무료 VM에 24시간 운영 환경 구축
Phase 5 ─ 운영 개선    (상시)   통계, 예외 처리, 편의 기능
```

---

## Phase 0 — 준비: 봇 계정 만들기

- [ ] [Discord Developer Portal](https://discord.com/developers/applications)에서 **New Application** 생성
- [ ] **Bot 탭** → 토큰 발급 (한 번만 보여주므로 즉시 복사, 절대 공유 금지)
- [ ] **Privileged Gateway Intents** 활성화:
  - [ ] `SERVER MEMBERS INTENT` — 멤버 목록 조회/추방에 필요
  - [ ] (`PRESENCE`, `MESSAGE CONTENT`는 불필요 — 끈 채로 둔다)
- [ ] **OAuth2 → URL Generator**에서 초대 링크 생성:
  - Scopes: `bot`, `applications.commands`
  - Bot Permissions: `View Channels`, `Send Messages`, `Kick Members`
- [ ] 생성된 URL로 스터디 서버에 봇 초대
- [ ] 개발자 모드 켜고(디스코드 설정 → 고급) 다음 ID 복사해두기:
  - 서버(길드) ID
  - 스터디 음성채널 ID
  - 출석 공지용 텍스트 채널 ID

**완료 기준**: 서버 멤버 목록에 봇이 (오프라인 상태로) 보인다.

## Phase 1 — 뼈대: 봇 접속과 이벤트 수신

- [ ] `npm install` (discord.js, better-sqlite3, node-cron, dotenv)
- [ ] `.env`에 토큰/ID 설정, `.gitignore`에 `.env`, `data/` 추가
- [ ] `src/index.js`: 클라이언트 생성, 로그인, `ready` 로그 출력
- [ ] `voiceStateUpdate` 이벤트 핸들러 연결 — 입장/퇴장 시 콘솔 로그만 찍기

**완료 기준**: 봇이 온라인으로 표시되고, 음성채널에 들어갔다 나오면 터미널에 로그가 찍힌다.

## Phase 2 — 출석 코어

- [ ] SQLite 스키마 생성 (`voice_segments`, `attendance`, `finalized_sessions`) — [ARCHITECTURE.md](ARCHITECTURE.md) 참고
- [ ] KST 기준 현재 시간대(오전/저녁/해당없음) 판별 함수
- [ ] 입장 시: 구간(segment) 열기 + 해당 시간대 **최초 입장이면** `🌅 출근` 공지
- [ ] 퇴장 시: 구간 닫기 + **10분 디바운스** 후 퇴근 공지 (10분 내 재입장하면 취소)
- [ ] 퇴근 공지 후 재입장 시 `재입장` 공지 (10분 내 복귀는 무공지)
- [ ] 시간대 시작 크론(09:00, 19:00): 이미 접속 중인 멤버 구간 열기
- [ ] 시간대 종료 크론(12:00, 22:00): 정산
  - 열린 구간 닫기 → 구간 합산 → 2시간 이상이면 출석 기록
  - 아직 퇴근 공지가 안 나간 참여자만 `🏁 퇴근` 공지 (접속 중 + 디바운스 대기 중)
- [ ] 봇 재시작 시 남아있는 미종료 구간 정리
- [ ] 정산 누락 복구 — 시작 시 `finalized_sessions`에 없는 과거 시간대를 늦게라도 정산

**완료 기준**: 테스트 시간대를 짧게 바꿔서(예: 5분 시간대 + 2분 기준) 출근 → 퇴근 → 출석 인정까지 전체 흐름이 동작한다.

## Phase 3 — 규율 자동화

- [ ] `warnings` 테이블 추가
- [ ] 주간 점검 크론: 멤버별 최근 7일 출석 일수 집계
  - 3회 미만 → 경고 +1, 공지 채널에 알림
  - 경고 3회 도달 → Kick + 알림
- [ ] 점검 제외 대상 처리: 봇, 운영진(관리 권한 보유자), **가입 7일 미만 신규 멤버**
- [ ] 추방(Kick) 시 해당 유저 경고 0으로 초기화 — 재입장 시 새 출발 (출석 이력은 보존)
- [ ] `/출석현황` 슬래시 명령어: 최근 7일 출석 일수, 경고 횟수 (본인에게만 표시)

**완료 기준**: 테스트 계정으로 경고 부여 → 3회 누적 시 추방까지 확인.

## Phase 4 — OCI 배포

상세 절차는 [DEPLOY-OCI.md](DEPLOY-OCI.md) 참고.

- [ ] Oracle Cloud 가입 (Always Free)
- [ ] VM 생성 (Ubuntu, Always Free 대상 인스턴스)
- [ ] Node.js 설치, 코드 업로드(git clone), `.env` 구성
- [ ] systemd 서비스 등록 — 부팅 시 자동 시작 + 죽으면 자동 재시작
- [ ] DB 파일 주기적 백업 (cron + 간단한 복사면 충분)

**완료 기준**: 내 PC를 꺼도 봇이 온라인이고, VM을 재부팅해도 봇이 스스로 살아난다.

## Phase 5 — 운영 개선 (아이디어)

- [x] 주간·월간 출석 랭킹 자동 공지 + 누적 출석 레벨(칭호·역할) — 완료
- [x] 특정 멤버 유예 처리 — `/유예 @멤버 일수`(자동 만료), `/유예해제`, `출석유예` 역할 지원 완료
- [ ] 운영진용 명령어: 경고 수동 조정
- [ ] 화면 공유 여부 감지 (`voiceState.streaming`) — 규칙상 필수이므로 리마인드 메시지
- [ ] 출석 데이터 CSV 내보내기
- [ ] 봇 상태 모니터링 (다운 시 운영진 DM 등)

# 명령어 사용법

봇의 슬래시 명령어 목록과 권한. 디스코드 채팅창에 `/`를 입력하면 자신이 쓸 수 있는 명령어만 자동완성에 뜬다.

## 권한 요약

| 명령어 | 누가 쓸 수 있나 | 응답 공개 범위 |
|---|---|---|
| `/출석현황` | **모든 멤버** | 본인에게만 보임 (ephemeral) |
| `/유예` | **운영진만** (멤버 추방 권한 보유자) | 실행한 채널에 공개 |
| `/유예해제` | **운영진만** (멤버 추방 권한 보유자) | 실행한 채널에 공개 |

- 권한은 명령어 등록 시 `setDefaultMemberPermissions`(멤버 추방 권한)로 강제된다 — 일반 멤버에게는 `/유예`가 **자동완성 목록에 아예 안 뜨고**, 강제로 호출할 방법도 없다
- 추가로 서버 설정 → 연동 → 봇에서 명령어별로 특정 역할/채널에만 허용하도록 덮어쓸 수도 있다 (기본값으로 충분해서 보통 불필요)

## /출석현황 — 내 출석 확인 (모두)

```
/출석현황
```

응답 예시 (본인에게만 보임):

```
📊 홍길동님의 출석 현황
- 이번 주(월~일) 출석: 2회 (기준: 3회 이상)
- 일요일까지 1회 더 필요해요
- 경고: 1/3
- 유예 중: 2026-07-20까지 점검 제외 🛌   ← 유예 등록된 경우에만 표시
```

## /유예 — 사정 있는 멤버 점검 제외 (운영진)

시험 기간·여행 등으로 한동안 참여가 어려운 멤버를 주간 점검에서 제외한다.

```
/유예 대상:@홍길동 일수:14
```

- `일수`: 오늘부터 며칠간 제외할지 (1~90)
- 기간이 지나면 **자동으로 점검 대상에 복귀** — 해제 명령을 따로 쓸 필요 없음
- **유예가 걸쳐 있던 주는 통째로 면제** — 예: 토요일에 유예가 끝나도 그 주 일요일 점검은 건너뛰고, 다음 주(온전한 주)부터 3회를 채우면 된다
- 같은 사람에게 다시 실행하면 기간이 **덮어써진다** (연장/단축 모두 이 방법으로)
- 유예 중에도 출석 기록은 정상 누적되고, 경고도 지워지지 않는다 (점검만 일시정지)
- 참고: `출석유예` **역할** 방식은 떼는 순간 바로 점검 대상이 되므로(뗀 시점을 시스템이 모름), 역할 해제는 일요일 점검 이후에 하거나 처음부터 `/유예`를 쓰는 게 안전하다

응답:
```
🛌 홍길동님을 2026-07-20까지 출석 점검에서 제외합니다.
유예가 걸친 주는 통째로 면제되고, 2026-07-27(월)부터 점검이 다시 적용돼요.
```
(점검 재개일을 봇이 자동 계산해서 알려주므로 애매할 게 없다)

## /유예해제 — 유예 조기 해제 (운영진)

예정보다 일찍 복귀했을 때 유예를 즉시 끝낸다.

```
/유예해제 대상:@홍길동
```

응답: `✅ 홍길동님의 출석 유예를 해제했습니다. 다음 주간 점검부터 다시 포함돼요.`

## 명령어 없이 되는 것들 (자동)

| 동작 | 트리거 |
|---|---|
| 출근/퇴근/재입장 공지 | 음성채널 입장·퇴장 (명령어 불필요) |
| 출석 인정 | 시간대 종료 정산 (12:00 / 22:00) |
| 경고/추방 | 주간 점검 (일요일 22:05) |
| 무기한 유예 | `출석유예` **역할**을 멤버에게 부여 (명령어 대신 역할로도 가능, 뗄 때까지 유지) |
| 스터디원 역할 자동 부여 | 새 멤버가 서버에 들어오면 봇이 `스터디원` 역할을 자동 배정 |
| 신입 환영 안내 | 입장 즉시 #환영-인사에 멘션으로 환영 + **출석 점검 시작일(월요일) 자동 계산 안내** — 7일 유예를 몰라서 불안해할 일 없음. `/출석현황`에도 적응 기간 중이면 표시 |
